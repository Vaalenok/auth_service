## Схема структуры управление ограничениями доступа
В базе данных создано 4 таблицы - users, roles, access_rules
и production_elements. Пользователь имеет роль, в которую 
уже вшиты правила доступа к каждому отдельному элементу. 
Элементы, так же как и правила доступа к ним, настраиваются вручную, 
позволяя легко и наглядно контролировать доступ определенной роли. 
Упор делал на гибкость системы и её масштабируемость.

- users -> MtoO -> roles
- roles -> OtoM -> access_rules
- access_rules -> MtoO -> production_elements

### Создание бд
1. Заполнить .env файл по образцу
2. Поднять контейнер ``docker run --name <название контейнера> -e POSTGRES_USER=<имя пользователя бд> -e POSTGRES_PASSWORD=<пароль пользователя бд> -e POSTGRES_DB=<название бд> -p 5432:5432 -d postgres:latest postgres``
